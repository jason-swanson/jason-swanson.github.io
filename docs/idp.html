idp

Tools for working with iterated Dirichlet processes.

Module description

Consider a situation in which there are several agents, all from the same population. Each agent undertakes a sequence of actions. These actions are chosen according to the agent's particular tendencies. Although different agents have different tendencies, there may be patterns in the population.

We observe a certain set of agents over a certain amount of time. Based on these observations, we want to make probabilistic forecasts about two things:

  • The future behavior of the agents we have observed.
  • The behavior of a new (unobserved) agent from the population.

An iterated Dirichlet process is a mathematical tool that can model this. The file, idp-paper-9.pdf, gives the mathematical details. (In this rest of this documentation, we refer to this file as the "Paper.") This module, idp, is a Python implementation of the iterated Dirichlet process (IDP).

Examples

Pressed penny machines

Imagine a pressed penny machine, like those found in museums or tourist attractions. For a fee, the machine presses a penny into a commemorative souvenir. Now imagine the machine is broken, so that it mangles all the pennies we feed it. Each pressed penny it creates is mangled in its own way. Each has its own probability of landing on heads when flipped. In this situation, the agents are the pennies and the actions are the heads and tails that they produce.

Now suppose we create seven mangled pennies and flip each one 5 times. Suppose that the first four coins each result in 4 heads out of 5 flips. The fifth coin results in 1 head, the sixth coin results in 4 heads, and the seventh coin results in 3 heads. If we flip the fifth coin one more time, what is the probability that it will land on heads? Looking only at the past flips of the fifth coin, it seems biased towards tails. Overall, though, the machine seems to make coins that are biased towards heads. All seven coins could be heads-biased, and the fifth coin just had an unlucky run. How should we balance these competing intuitions to arrive at a probability?

One way to answer this is to model the situation with an IDP. We can do this with the IDPModel class as follows:

>>> from idp import IDPModel
>>> idpModel = IDPModel(1, 1, 0.5,
...     [[5 - x, x] for x in [4, 4, 4, 4, 1, 4, 3]])

Next, we run 10,000 weighted simulations and use them to estimate the probability that the fifth coin, if flipped again, will land on heads:

>>> idpModel.addWeightedSims(10000)
>>> idpModel.postProb(4)
0.44196003938121636

Likewise, we can estimate the same probability for the first and seventh coins:

>>> idpModel.postProb(0)
0.7236483548829938
>>> idpModel.postProb(6)
0.6806816746246023

We can also estimate the probability that a new coin from the machine will land on heads when flipped. For this, we can use any argument larger than the highest index in our observed data.

>>> idpModel.postProb(7)
0.6559370622554825
>>> idpModel.postProb(47586)
0.6559370622554825

If the new coin from the machine has probability of heads $\theta$, then $\theta$ is a random variable taking values in $[0, 1]$. The following will plot the cumulative distribution function of $\theta$:

>>> import numpy as np
>>> import matplotlib.pyplot as plt
>>> t = np.linspace(0, 1, 200)
>>> plt.plot(t, idpModel.meanRDGcdf(t))
>>> plt.show()

The random variable $\theta$ has a discrete component, so it does not have a probability density function. But we can plot an approximate density as follows:

>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> plt.show()

The approximate density is computed by smoothing out the discrete point masses. The degree of smoothing is determined by a parameter called the "bandwidth". (See IDPModel.meanRDGpdf for details.) This bandwidth is computed automatically. We can view the bandwidth that was used most recently, and we can also set the bandwidth manually.

>>> idpModel.bandwidth
np.float64(0.11679260408842912)
>>> plt.plot(t, idpModel.meanRDGpdf(t, bandwidth=0.001))
>>> plt.show()

Flipping thumbtacks

Imagine a box of 320 thumbtacks. We flick each thumbtack 9 times. If it lands point up, we call it a success. Point down is a failure. Because of the imperfections, each thumbtack has its own probability of success. The file, tacks.csv, contains the success and failure counts for this experiment. The data in tacks.csv is exactly the data considered in Liu 1996. (This data originally came from an experiment by Beckett and Diaconis in 1994. In the original experiment, there were not 320 thumbtacks. Rather, there were 16 thumbtacks, 2 flickers, and 10 surfaces.)

To study this situation, we first create the model and perform 10,000 simulations. We then look at the approximate density of the posterior probability of success.

>>> import csv
>>> with open('tacks.csv') as f:
...     sheet = list(csv.reader(f))
...     rowCounts = [[int(value) for value in row] for row in sheet]
>>> idpModel = IDPModel(1, 2, 0.5, rowCounts)
>>> idpModel.addWeightedSims(10000) # took ~3m on a Macbook
>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> idpModel.bandwidth
0.11170697606064083
>>> plt.show()

Here, we use a row concentration parameter of $2$ and a base measure that assigns probability $1/2$ to both $0$ and $1$. This matches the distribution used by Liu. What we call the column concentration parameter is denoted by $c$ in Liu's analysis. Liu considers the cases $c = 0.1$, $c = 1$, $c = 5$, and $c = 10$. The above image is our result for $c = 1$. For $c = 10$, we have the following:

>>> idpModel = IDPModel(10, 2, 0.5, rowCounts)
>>> idpModel.addWeightedSims(10000) # took ~3m on a Macbook
>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> idpModel.bandwidth
0.10303529320118705
>>> plt.show()

These graphs look about the same as Liu's, but with minor differences. It is difficult, though, to make a direct comparison. Liu also uses Gaussian kernel smoothing, but does not provide any details about it. For example, we do not know the bandwidth used to produce Liu's graphs.

306def _betaFuncMult(alpha):
307    '''Computes and returns the value of the multivariate beta function,
308    evaluated at the given list of arguments.
309
310    **Arguments:**
311
312    * **alpha** (`list`[`float`]): The list of arguments for the
313        multivariate beta function.
314
315    '''
316    return np.exp(np.sum(gammaln(alpha)) - gammaln(np.sum(alpha)))
317
318class IDPModel:
319    r'''An IDP model on the state space $S = \\{0, ..., L - 1\\}$.
320
321    The IDP model consists of both an IDP and some observed data. See
322    Section 9 of the Paper for details.
323
324    '''
325
326    def __init__(self, colConc, rowConc, baseMeas, rowCounts):
327        '''The constructor first uses the given arguments to set the
328        corresponding properties. If a property returns a tuple or a
329        tuple of tuples, then the argument can be passed any
330        `array-like` object, such as a list or a list of lists.
331
332        For the `baseMeas` property, the corresponding argument can be
333        passed a single floating point value `p`, in which case the
334        property will be set to `(1 - p, p)`.
335
336        '''
337        # `baseMeas` and `rowCounts` are stored as ndarrays rather than
338        # lists, since vector addition fails for lists but works for
339        # ndarrays. This is important in the constructor of the
340        # `WeightedSim` class.
341        self._scale = 0
342
343        if isinstance(baseMeas, float):
344            baseMeas = [1 - baseMeas, baseMeas]
345        baseMeas = np.array(baseMeas)
346        rowCounts = np.array(rowCounts)
347
348        self._idp = {
349            'colConc': colConc,
350            'rowConc': rowConc,
351            'baseMeas': baseMeas
352        }
353
354        self._rowCounts = rowCounts
355
356        self._likelihoods = []
357        self._weightedSims = []
358        self._bandwidth = None
359
360    @property
361    def colConc(self):
362        r'''`float`: The column concentration parameter. Should be
363        positive. Denoted by $\kappa$ in Section 9 of the Paper.
364        '''
365        return self._idp['colConc']
366
367    @property
368    def rowConc(self):
369        r'''`float`: The row concentration parameter. Should be
370        positive. Denoted by $\varepsilon$ in Section 9 of the Paper.
371        '''
372        return self._idp['rowConc']
373
374    @property
375    def baseMeas(self):
376        r'''`tuple`(`float`): The base probability vector. Should be a
377        list of positive numbers that sum to 1. Denoted by
378        $\boldsymbol{p} = (p_0, \ldots, p_{L - 1})$ in Section 9 of the
379        Paper, where $p_\ell = \varrho(\{\ell\})$.
380        '''
381        return tuple(self._idp['baseMeas'])
382
383    @property
384    def rowCounts(self):
385        r'''`tuple`(`tuple`(`int`)): The row counts for the observed
386        data. If `row` is $m - 1$ and `state` is $\ell$, then
387        `rowCounts`[`row`][`state`] is denoted by $\overline{y}_{m\ell}$
388        in Section 9 of the Paper. It is the number of observed
389        instances of `state` in the given row of data.
390        '''
391        return tuple(tuple(row) for row in self._rowCounts)
392
393    @property
394    def likelihoods(self):
395        r'''`tuple`(`float`): The prior likelihoods. If `row` equals
396        $m - 1$, then `likelihoods`[`row`] is denoted by
397        $\varrho_N(A_m)$ in Section 9 of the Paper. It is the prior
398        probability of seeing the row counts for the given row. This
399        property is computed by the constructor, and should not be
400        changed.
401        '''
402        if len(self._likelihoods) < len(self._rowCounts):
403            rowConc = self._idp['rowConc']
404            baseMeas = self._idp['baseMeas']
405            denom = _betaFuncMult(rowConc * baseMeas)
406            for row in self._rowCounts[len(self._likelihoods):]:
407                self._likelihoods.append(
408                    _betaFuncMult(rowConc * baseMeas + row) / denom
409                )
410        return tuple(self._likelihoods)
411
412    @property
413    def scale(self):
414        r'''`float`: The scale factor to use for weighted simulations.
415        Denoted by $\log c$ in Section 9 of the Paper. Defaults to 0.
416        Altering this value runs the `WeightedSim.updateScale` method on
417        every simulation in the `weightedSims` property. If a
418        `ValueError` occurs in this process, then no changes will be
419        made, neither to the `scale` property, nor to any of the
420        simulations in the `weightedSims` property.
421        '''
422        return self._scale
423
424    @scale.setter
425    def scale(self, value):
426        oldVal = self._scale
427        self._scale = value
428        try:
429            for weightedSim in self._weightedSims:

514    def postProb(self, row, state=1):
515        r'''Computes and returns the posterior probability of seeing the
516        given state on a new trial in the given row. If `row` is
517        $m - 1$, `state` is $\ell$ , and the length of `rowCounts` is
518        $M$, then the value returned by this method is denoted in the
519        Paper by $P(\xi_{m, N_m + \ell} = 1 \mid Y = y)$ in the case
520        $m \le M$, and by $P(\xi_{M + 1, 1} = \ell \mid Y = y)$ in the
521        case $m > M$. This method implements (9.4) and (9.5) in the
522        Paper.
523
524        **Arguments:**
525
526        * **row** (`int`): The index of the row whose posterior
527            probability should be computed.
528        * **state** (`int`): The state whose posterior probability
529            should be computed.
530
531        **Returns:**
532
533        * `float`: The posterior probability of seeing the given state.
534
535        **Raises:**
536
537        * **ValueError**: If the `weightedSims` property is empty.
538
539        '''
540        if not self._weightedSims:
541            raise ValueError('No weighted simulations found')
542
543        numRows = len(self._rowCounts)
544        if row < numRows:
545            return self._postProbLow(row, state)
546
547        colConc = self._idp['colConc']
548        baseMeas = self._idp['baseMeas']
549        return (
550            colConc * baseMeas[state]
551            + sum(self._postProbLow(m, state) for m in range(numRows))
552        ) / (colConc + numRows)

554    def _postProbLow(self, row, state=1):
555        '''This method is the same as `postProb`, except it requires
556        `row` to be less than the length of `rowCounts`.
557
558        '''
559        return sum(
560            weightedSim.totalWeight * weightedSim.rowDistSims[row][state]
561            for weightedSim in self._weightedSims
562        ) / sum(
563            weightedSim.totalWeight
564            for weightedSim in self._weightedSims

567    def meanRDGcdf(self, t, state=1):
568        r'''The mean of the row distribution generator (RDG) for a state
569        $\ell$ is a probability distribution on $[0, 1]$. More
570        specifically, using the notation in the Paper, it is the
571        conditional distribution of $\theta_{M + 1, \ell}$ given
572        $Y = y$, and an expression for it is given in (9.6) in the
573        Paper.
574
575        This method evaluates the cumulative distribution function (cdf)
576        of the mean of the RDG for the given state at the given point.
577        That is, it gives the cdf for the random probability of seeing
578        the given state in a new trial.
579
580        **Arguments:**
581
582        * **t** (`float`): The point at which to evaluate the cdf.
583        * **state** (`int`): The state for which the cdf is to be
584            computed.
585
586        **Returns:**
587
588        * `float`: The value of the cdf at the given point.
589
590        '''
591        colConc = self._idp['colConc']
592        numRows = len(self._rowCounts)
593
594        sample, weights = self._makeSample(state)
595        sampleCDF = ECDFDiscrete(sample, weights)
596
597        return (
598            (colConc / (colConc + numRows)) * self._prior(state).cdf(t) +
599            (numRows / (colConc + numRows)) * sampleCDF(t)
600        )

602    def _prior(self, state=None):
603        '''Returns the prior distribution of the mean RDG for the given
604        state. If no state is given, returns the prior of a row
605        distribution.
606
607        '''
608        if state is None:
609            return dirichlet(self._idp['rowConc'] * self._idp['baseMeas'])
610        return beta(
611            self._idp['rowConc'] * self._idp['baseMeas'][state],
612            self._idp['rowConc'] * (1 - self._idp['baseMeas'][state])
613        )

615    def _makeSample(self, state=None):
616        '''Creates and returns a weighted list of simulated row
617        distributions evaluated at the given state. If no state is
618        given, returns the row distributions themselves. Used to create
619        the empirical distribution of the mean RDG and to simulate a new
620        row distribution.
621
622        '''
623        # The `sample` vector is ordered as
624        # \[
625        #   \theta_{1 \ell}^{*, 1}, \ldots, \theta_{M \ell}^{*, 1},
626        #   \theta_{1 \ell}^{*, 2}, \ldots, \theta_{M \ell}^{*, 2},
627        #   \ldots,
628        #   \theta_{1 \ell}^{*, K}, \ldots, \theta_{M \ell}^{*, K}
629        # \]
630        numRows = len(self._rowCounts)
631        sample = np.array([
632            weightedSim.rowDistSims[m] if state is None
633            else weightedSim.rowDistSims[m][state]
634            for weightedSim in self._weightedSims # outer loop
635            for m in range(numRows)  # inner loop
636        ])
637        weights = np.concatenate([
638            [weightedSim.totalWeight] * numRows
639            for weightedSim in self._weightedSims
640        ])
641        return sample, weights

643    def meanRDGpdf(self, t, state=1, bandwidth=None):
644        r'''This method evaluates an approximate density (pdf) for the
645        mean of the RDG for the given state. (See `meanRDGcdf` for an
646        explanation of the RDG.) The mean of the RDG has a discrete
647        component, and therefore has no density. This method creates an
648        approximate density using Gaussian kernel density estimation
649        (KDE). More specifically, it treats the point mass $\delta_x$ as
650        having a Gaussian density with mean $x$ and standard deviation
651        $h$, where $h$ is the "bandwidth" of the estimate.
652
653        **Arguments:**
654
655        * **t** (`float`): The point at which to evaluate the
656            approximate pdf.
657        * **state** (`int`): The state for which the pdf is to be
658            computed.
659        * **bandwidth** (`float`): The bandwidth to use in the Gaussian
660            KDE. If omitted, the bandwidth will be calculated using
661            Scott's Rule. The calculated value can be retrieved using
662            the `bandwidth` property. This argument is passed directly
663            to the `gaussian_kde` function in the `scipy.stats` module.
664            See that documentation for further details.
665
666        **Returns:**
667
668        * `float`: The value of the approximate pdf at the given point.
669
670        '''
671        colConc = self._idp['colConc']
672        numRows = len(self._rowCounts)
673
674        sample, weights = self._makeSample(state)
675        samplePDF = gaussian_kde(sample, bandwidth, weights)
676        self._bandwidth = samplePDF.factor
677
678        return (
679            (colConc / (colConc + numRows)) * self._prior(state).pdf(t) +
680            (numRows / (colConc + numRows)) * samplePDF(t)
681        )

688        choice = choices(

690            weights=[self._idp['colConc'], len(self._rowCounts)]
691        )[0]
692        if choice == 0:
693            return self._prior().rvs()[0]
694        sample, weights = self._makeSample()
695        return choices(sample, weights=weights)[0]

748    def updateRowCounts(self):
749        '''Updates the `rowWeights`, `rowDistSims`, and `totalWeight`
750        properties, if necessary, so that the number of rows in
751        `rowDistSims` is the same as the number of rows in the
752        `IDPModel.rowCounts` property of the associated `idpModel`
753        instance.
754
755        **Raises:**
756
757        * **ValueError**: If the computed value for the `totalWeight`
758            property is so small or so large that it evaluates to `0.0`
759            or `np.inf`.
760
761        '''
762        # If $L$ is the number of states in the IDP, then the case
763        # $L = 2$ is separated from the case $2 < L < \infty$ to speed
764        # up code execution.
765        baseMeas = np.array(self._idpModel.baseMeas)
766        numStates = len(baseMeas)
767
768        # expand properties if row counts have been added
769        numOldSims = len(self._rowDistSims)
770        for _, counts in enumerate(self._idpModel.rowCounts[numOldSims:]):
771            m = numOldSims + _
772
773            # build `rowWeights[m]`
774            self._rowWeights.append([])
775            for i in range(m):
776                if numStates == 2:
777                    self._rowWeights[m].append(
778                        self._rowDistSims[i][0] ** counts[0] *
779                        self._rowDistSims[i][1] ** counts[1]
780                    )
781                else:
782                    weight = self._rowDistSims[i][0] ** counts[0]
783                    for state in range(1, numStates):
784                        weight *= self._rowDistSims[i][state] ** counts[state]
785                    self._rowWeights[m].append(weight)
786            self._rowWeights[m].append(
787                self._idpModel.colConc * self._idpModel.likelihoods[m]
788            )
789
790            # build `rowDistSims[m]`
791            choice = choices(range(m + 1), weights=self._rowWeights[m])[0]
792            if choice < m:
793                self._rowDistSims.append(self._rowDistSims[choice])
794            else:
795                if numStates == 2:
796                    rowDistSim = beta.rvs(
797                        self._idpModel.rowConc * baseMeas[1] + counts[1],
798                        self._idpModel.rowConc * baseMeas[0] + counts[0]
799                    )
800                    self._rowDistSims.append([1 - rowDistSim, rowDistSim])
801                else:
802                    self._rowDistSims.append(
803                        # vector sum fails for lists, works for ndarray
804                        dirichlet.rvs(
805                            self._idpModel.rowConc * baseMeas + counts
806                        )[0]
807                    )
808
809            # build `totalWeight`
810            #
811            # The m-th iteration of the `for` loop multiplies the
812            # previous value of `totalWeight` by the (m + 1)-th factor
813            # in the definition of $V$ in Section 9 of the Paper.
814            self._logWeight += (
815                self._idpModel.scale +
816                np.log(m + 1) -
817                np.log(self._idpModel.colConc + m) +
818                np.log(sum(self._rowWeights[m]))
819            )
820
821        # check that total weight is not out of bounds
822        if np.exp(self._logWeight) in (0, np.inf):
823            adj = -self._logWeight/len(self._idpModel.rowCounts)
824            raise ValueError(
825                'Simulation weight out of bounds.\n' +
826                f'Try adjusting scale factor by {adj}'
827            )
828
829        # trim properties if row counts have been removed
830        if numOldSims > len(self._idpModel.rowCounts):
831            for m in range(
832                numOldSims - 1, len(self._idpModel.rowCounts) - 1, -1
833            ):
834                self._logWeight -= (
835                    self._idpModel.scale +
836                    np.log(m + 1) -
837                    np.log(self._idpModel.colConc + m) +
838                    np.log(sum(self._rowWeights[m]))
839                )
840                del self._rowWeights[-1]
841                del self._rowDistSims[-1]

844        '''Updates the `totalWeight` property, if necessary, to account
845        for a change in the `IDPModel.scale` property of the associated
846        `idpModel` instance.

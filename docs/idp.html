idp

Tools for working with iterated Dirichlet processes.

Module description

Consider a situation in which there are several agents, all from the same population. Each agent undertakes a sequence of actions. These actions are chosen according to the agent's particular tendencies. Although different agents have different tendencies, there may be patterns in the population.

We observe a certain set of agents over a certain amount of time. Based on these observations, we want to make probabilistic forecasts about two things:

  • The future behavior of the agents we have observed.
  • The behavior of a new (unobserved) agent from the population.

An iterated Dirichlet process is a mathematical tool that can model this. The file, idp-paper-9.pdf, gives the mathematical details. (In this rest of this documentation, we refer to this file as the "Paper.") This module, idp, is a Python implementation of the iterated Dirichlet process (IDP).

Examples

Pressed penny machines

Imagine a pressed penny machine, like those found in museums or tourist attractions. For a fee, the machine presses a penny into a commemorative souvenir. Now imagine the machine is broken, so that it mangles all the pennies we feed it. Each pressed penny it creates is mangled in its own way. Each has its own probability of landing on heads when flipped. In this situation, the agents are the pennies and the actions are the heads and tails that they produce.

Now suppose we create seven mangled pennies and flip each one 5 times. Suppose that the first four coins each result in 4 heads out of 5 flips. The fifth coin results in 1 head, the sixth coin results in 4 heads, and the seventh coin results in 3 heads. If we flip the fifth coin one more time, what is the probability that it will land on heads? Looking only at the past flips of the fifth coin, it seems biased towards tails. Overall, though, the machine seems to make coins that are biased towards heads. All seven coins could be heads-biased, and the fifth coin just had an unlucky run. How should we balance these competing intuitions to arrive at a probability?

One way to answer this is to model the situation with an IDP. We can do this with the IDPModel class as follows:

>>> from idp import IDPModel
>>> idpModel = IDPModel(1, 1, 0.5,
...     [[5 - x, x] for x in [4, 4, 4, 4, 1, 4, 3]])

Next, we run 10,000 weighted simulations and use them to estimate the probability that the fifth coin, if flipped again, will land on heads:

>>> idpModel.addWeightedSims(10000)
>>> idpModel.postProb(4)
np.float64(0.43811168787852106)

Likewise, we can estimate the same probability for the first and seventh coins:

>>> idpModel.postProb(0)
np.float64(0.7248327477323794)
>>> idpModel.postProb(6)
np.float64(0.6800744641793576)

We can also estimate the probability that a new coin from the machine will land on heads when flipped. For this, we can use any argument larger than the highest index in our observed data.

>>> idpModel.postProb(7)
np.float64(0.6556091734873866)
>>> idpModel.postProb(47586)
np.float64(0.6556091734873866)

If the new coin from the machine has probability of heads $\theta$, then $\theta$ is a random variable taking values in $[0, 1]$. The following will plot the cumulative distribution function of $\theta$:

>>> import numpy as np
>>> import matplotlib.pyplot as plt
>>> t = np.linspace(0, 1, 200)
>>> plt.plot(t, idpModel.meanRDGcdf(t))
>>> plt.show()

The random variable $\theta$ has a discrete component, so it does not have a probability density function. But we can plot an approximate density as follows:

>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> plt.show()

The approximate density is computed by smoothing out the discrete point masses. The degree of smoothing is determined by a parameter called the "bandwidth". (See IDPModel.meanRDGpdf for details.) This bandwidth is computed automatically. We can view the bandwidth that was used most recently, and we can also set the bandwidth manually.

>>> idpModel.bandwidth
np.float64(0.11678046360237249)
>>> plt.plot(t, idpModel.meanRDGpdf(t, bandwidth=0.001))
>>> plt.show()

Flipping thumbtacks

Imagine a box of 320 thumbtacks. We flick each thumbtack 9 times. If it lands point up, we call it a success. Point down is a failure. Because of the imperfections, each thumbtack has its own probability of success. The file, tacks.csv, contains the success and failure counts for this experiment. The data in tacks.csv is exactly the data considered in Liu 1996. (This data originally came from an experiment by Beckett and Diaconis in 1994. In the original experiment, there were not 320 thumbtacks. Rather, there were 16 thumbtacks, 2 flickers, and 10 surfaces.)

To study this situation, we first create the model and perform 10,000 simulations. We then look at the approximate density of the posterior probability of success.

>>> import csv
>>> with open('tacks.csv') as f:
...     sheet = list(csv.reader(f))
...     rowCounts = [[int(value) for value in row] for row in sheet]
>>> idpModel = IDPModel(1, 2, 0.5, rowCounts)
>>> idpModel.addWeightedSims(10000) # took ~3m on a Macbook
>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> idpModel.bandwidth
np.float64(0.10937162488547277)
>>> plt.show()

Here, we use a row concentration parameter of $2$ and a base measure that assigns probability $1/2$ to both $0$ and $1$. This matches the distribution used by Liu. What we call the column concentration parameter is denoted by $c$ in Liu's analysis. Liu considers the cases $c = 0.1$, $c = 1$, $c = 5$, and $c = 10$. The above image is our result for $c = 1$. For $c = 10$, we have the following:

>>> idpModel = IDPModel(10, 2, 0.5, rowCounts)
>>> idpModel.addWeightedSims(10000) # took ~3m on a Macbook
>>> plt.plot(t, idpModel.meanRDGpdf(t))
>>> idpModel.bandwidth
np.float64(0.0942997015794818)
>>> plt.show()

These graphs look about the same as Liu's, but with minor differences. It is difficult, though, to make a direct comparison. Liu also uses Gaussian kernel smoothing, but does not provide any details about it. For example, we do not know the bandwidth used to produce Liu's graphs.

Amazon reviews

Liu 1996 only covers agents with two possible actions, such as coins and thumbtacks. The IDP, though, can handle agents whose range of possible actions is arbitrary.

196def _betaFuncMult(alpha):
197    '''Computes and returns the value of the multivariate beta function,
198    evaluated at the given list of arguments.
199
200    **Arguments:**
201
202    * **alpha** (`list`[`float`]): The list of arguments for the
203        multivariate beta function.
204
205    '''
206    return np.exp(np.sum(gammaln(alpha)) - gammaln(np.sum(alpha)))

208class IDPModel:
209    r'''An IDP model on the state space $S = \\{0, ..., L - 1\\}$.
210
211    The IDP model consists of both an IDP and some observed data. See
212    Section 9 of the Paper for details.
213
214    '''
215
216    def __init__(self, colConc, rowConc, baseMeas, rowCounts):
217        '''The constructor first uses the given arguments to set the
218        corresponding properties. If a property returns a tuple (of
219        tuples), then the argument should be passed a list (of lists).
220
221        For the `baseMeas` property, the corresponding argument can be
222        passed a single floating point value `p`, in which case the
223        property will be set to `(1 - p, p)`.
224
225        '''
226        # `baseMeas` and `rowCounts` are stored as ndarrays rather than
227        # lists, since vector addition fails for lists but works for
228        # ndarrays. This is important in the constructor of the
229        # `WeightedSim` class.
230        if isinstance(baseMeas, float):
231            baseMeas = [1 - baseMeas, baseMeas]
232        baseMeas = np.array(baseMeas)
233        rowCounts = np.array(rowCounts)
234
235        self._idp = {
236            'colConc': colConc,
237            'rowConc': rowConc,
238            'baseMeas': baseMeas
239        }
240
241        self._rowCounts = rowCounts
242
243        self._likelihoods = []
244        denom = _betaFuncMult(rowConc * baseMeas)
245        for row in rowCounts:
246            self._likelihoods.append(
247                _betaFuncMult(rowConc * baseMeas + row) / denom
248            )
249
250        self._weightedSims = []
251
252        self._bandwidth = None

255    def colConc(self):
256        r'''`float`: The column concentration parameter. Should be
257        positive. Denoted by $\kappa$ in Section 9 of the Paper.
258        '''
259        return self._idp['colConc']
260
261    @property
262    def rowConc(self):
263        r'''`float`: The row concentration parameter. Should be
264        positive. Denoted by $\varepsilon$ in Section 9 of the Paper.
265        '''
266        return self._idp['rowConc']
267
268    @property
269    def baseMeas(self):
270        r'''`tuple`(`float`): The base probability vector. Should be a
271        list of nonnegative numbers that sum to 1. Denoted by
272        $\boldsymbol{p} = (p_0, \ldots, p_{L - 1})$ in Section 9 of the
273        Paper, where $p_\ell = \varrho(\{\ell\})$.
274        '''
275        return tuple(self._idp['baseMeas'])
276
277    @property
278    def rowCounts(self):
279        r'''`tuple`(`tuple`(`int`)): The row counts for the observed
280        data. If `row` is $m - 1$ and `state` is $\ell$, then
281        `rowCounts`[`row`][`state`] is denoted by $\overline{y}_{m\ell}$
282        in Section 9 of the Paper. It is the number of observed
283        instances of `state` in the given row of data.
284        '''
285        return tuple(tuple(row) for row in self._rowCounts)
286
287    @property
288    def likelihoods(self):
289        r'''`tuple`(`float`): The prior likelihoods. If `row` equals
290        $m - 1$, then `likelihoods`[`row`] is denoted by
291        $\varrho_N(A_m)$ in Section 9 of the Paper. It is the prior
292        probability of seeing the row counts for the given row. This
293        property is computed by the constructor, and should not be
294        changed.
295        '''
296        return tuple(self._likelihoods)

336    def postProb(self, row, state=1):
337        r'''Computes and returns the posterior probability of seeing the
338        given state on a new trial in the given row. If `row` is
339        $m - 1$, `state` is $\ell$ , and the length of `rowCounts` is
340        $M$, then the value returned by this method is denoted in the
341        Paper by $P(\xi_{m, N_m + \ell} = 1 \mid Y = y)$ in the case
342        $m \le M$, and by $P(\xi_{M + 1, 1} = \ell \mid Y = y)$ in the
343        case $m > M$. This method implements (9.4) and (9.5) in the
344        Paper.
345
346        **Arguments:**
347
348        * **row** (`int`): The index of the row whose posterior
349            probability should be computed.
350        * **state** (`int`): The state whose posterior probability
351            should be computed.
352
353        **Returns:**
354
355        * `float`: The posterior probability of seeing the given state.
356
357        **Raises:**
358
359        * **ValueError**: If the `weightedSims` property is empty.
360
361        '''
362        if not self._weightedSims:
363            raise ValueError('No weighted simulations found')
364
365        numRows = len(self._rowCounts)
366        if row < numRows:
367            return self._postProbLow(row, state)
368
369        colConc = self._idp['colConc']
370        baseMeas = self._idp['baseMeas']
371        return (
372            colConc * baseMeas[state]
373            + sum(self._postProbLow(m, state) for m in range(numRows))
374        ) / (colConc + numRows)
375
376    def _postProbLow(self, row, state=1):
377        '''This method is the same as `postProb`, except it requires
378        `row` to be less than the length of `rowCounts`.
379
380        '''
381        return sum(
382            weightedSim.totalWeight * weightedSim.rowDistSims[row][state]
383            for weightedSim in self._weightedSims
384        ) / sum(
385            weightedSim.totalWeight
386            for weightedSim in self._weightedSims
387        )

389    def meanRDGcdf(self, t, state=1):
390        r'''The mean of the row distribution generator (RDG) for a state
391        $\ell$ is a probability distribution on $[0, 1]$. More
392        specifically, using the notation in the Paper, it is the
393        conditional distribution of $\theta_{M + 1, \ell}$ given
394        $Y = y$, and an expression for it is given in (9.6) in the
395        Paper.
396
397        This method evaluates the cumulative distribution function (cdf)
398        of the mean of the RDG for the given state at the given point.
399        That is, it gives the cdf for the random probability of seeing
400        the given state in a new trial.
401
402        **Arguments:**
403
404        * **t** (`float`): The point at which to evaluate the cdf.
405        * **state** (`int`): The state for which the cdf is to be
406            computed.
407
408        **Returns:**
409
410        * `float`: The value of the cdf at the given point.
411
412        '''
413        colConc = self._idp['colConc']
414        numRows = len(self._rowCounts)
415
416        sample, weights = self._makeSample(state)
417        sampleCDF = ECDFDiscrete(sample, weights)
418
419        return (
420            (colConc / (colConc + numRows)) * self._prior(state).cdf(t) +
421            (numRows / (colConc + numRows)) * sampleCDF(t)
422        )
423
424    def _prior(self, state):
425        '''Returns the prior distribution of the mean RDG for the given
426        state.
427        '''
428        return beta(
429            self._idp['rowConc'] * self._idp['baseMeas'][state],
430            self._idp['rowConc'] * (1 - self._idp['baseMeas'][state])
431        )
432
433    def _makeSample(self, state):
434        '''Creates and returns a weighted list of simulated row
435        distributions evaluated at the given state. Used to create the
436        empirical distribution of the mean RDG.
437        '''
438        # The `sample` vector is ordered as
439        # \[
440        #   \theta_{1 \ell}^{*, 1}, \ldots, \theta_{M \ell}^{*, 1},
441        #   \theta_{1 \ell}^{*, 2}, \ldots, \theta_{M \ell}^{*, 2},
442        #   \ldots,
443        #   \theta_{1 \ell}^{*, K}, \ldots, \theta_{M \ell}^{*, K}
444        # \]
445        numRows = len(self._rowCounts)
446        sample = np.array([
447            weightedSim.rowDistSims[m][state]
448            for weightedSim in self._weightedSims # outer loop
449            for m in range(numRows)  # inner loop
450        ])
451        weights = np.concatenate([
452            [weightedSim.totalWeight] * numRows
453            for weightedSim in self._weightedSims
454        ])
455        return sample, weights
456
457    def meanRDGpdf(self, t, state=1, bandwidth=None):
458        r'''This method evaluates an approximate density (pdf) for the
459        mean of the RDG for the given state. (See `meanRDGcdf` for an
460        explanation of the RDG.) The mean of the RDG has a discrete
461        component, and therefore has no density. This method creates an
462        approximate density using Gaussian kernel density estimation
463        (KDE). More specifically, it treats the point mass $\delta_x$ as
464        having a Gaussian density with mean $x$ and standard deviation
465        $h$, where $h$ is the "bandwidth" of the estimate.
466
467        **Arguments:**
468
469        * **t** (`float`): The point at which to evaluate the
470            approximate pdf.
471        * **state** (`int`): The state for which the pdf is to be
472            computed.
473        * **bandwidth** (`float`): The bandwidth to use in the Gaussian
474            KDE. If omitted, the bandwidth will be calculated using
475            Scott's Rule. The calculated value can be retrieved using
476            the `bandwidth` property. This argument is passed directly
477            to the `gaussian_kde` function in the `scipy.stats` module.
478            See that documentation for further details.
479
480        **Returns:**
481
482        * `float`: The value of the approximate pdf at the given point.
483
484        '''
485        colConc = self._idp['colConc']
486        numRows = len(self._rowCounts)
487
488        sample, weights = self._makeSample(state)
489        samplePDF = gaussian_kde(sample, bandwidth, weights)
490        self._bandwidth = samplePDF.factor
491
492        return (
493            (colConc / (colConc + numRows)) * self._prior(state).pdf(t) +
494            (numRows / (colConc + numRows)) * samplePDF(t)
495        )

497class WeightedSim:
498    r'''A weighted simulation for an `IDPModel` instance.
499
500    In the Paper, a weighted simulation is determined by the pair $(t,
501    \boldsymbol{\theta}_M^*)$ and has total weight $V$.
502
503    '''
504
505    def __init__(self, idpModel):
506        '''Uses the given argument to set the corresponding property,
507        then recursively computes and simulates the remaining
508        properties.
509
510        '''
511        # If $L$ is the number of states in the IDP, then the case
512        # $L = 2$ is separated from the case $2 < L < \infty$ to speed
513        # up code execution.
514        self._idpModel = idpModel
515        self._rowWeights = []
516        self._rowDistSims = []
517        self._totalWeight = 1
518
519        colConc = idpModel.colConc
520        rowConc = np.array(idpModel.rowConc)
521        baseMeas = np.array(idpModel.baseMeas)
522        numStates = len(baseMeas)
523        likelihoods = idpModel.likelihoods
524
525        for m, counts in enumerate(idpModel.rowCounts):
526            # build `rowWeights[m]`
527            self._rowWeights.append([])
528            for i in range(m):
529                if numStates == 2:
530                    self._rowWeights[m].append(
531                        self._rowDistSims[i][0] ** counts[0] *
532                        self._rowDistSims[i][1] ** counts[1]
533                    )
534                else:
535                    weight = self._rowDistSims[i][0] ** counts[0]
536                    for state in range(1, numStates):
537                        weight *= self._rowDistSims[i][state] ** counts[state]
538                    self._rowWeights[m].append(weight)
539            self._rowWeights[m].append(colConc * likelihoods[m])
540
541            # build `rowDistSims[m]`
542            choice = choices(range(m + 1), weights=self._rowWeights[m])[0]
543            if choice < m:
544                self._rowDistSims.append(self._rowDistSims[choice])
545            else:
546                if numStates == 2:
547                    rowDistSim = beta.rvs(
548                        rowConc * baseMeas[1] + counts[1],
549                        rowConc * baseMeas[0] + counts[0]
550                    )
551                    self._rowDistSims.append([1 - rowDistSim, rowDistSim])
552                else:
553                    self._rowDistSims.append(
554                        # vector sum fails for lists, works for ndarray
555                        dirichlet.rvs(rowConc * baseMeas + counts)[0]
556                    )
557
558            # build `totalWeight`
559            #
560            # The m-th iteration of the `for` loop multiplies the
561            # previous value of `totalWeight` by the (m + 1)-th factor
562            # in the definition of $V$ in Section 9 of the Paper.
563            self._totalWeight *= (
564                sum(self._rowWeights[m]) * (m + 1) / (colConc + m)
565            )
566
567    @property
568    def idpModel(self):
569        '''`IDPModel`: The IDP model for which the weighted simulation
570        is being created.
571        '''
572        return self._idpModel
573
574    @property
575    def rowWeights(self):
576        '''`tuple`(`tuple`(`float`)): The triangular array of row
577        weights for the weighted simulation. Denoted by $t$ in Section 9
578        of the Paper.
579        '''
580        return tuple(tuple(row) for row in self._rowWeights)
581
582    @property
583    def rowDistSims(self):
584        r'''`tuple`(`tuple`(`float`)): A tuple of simulated row
585        distributions. If `row` is $m - 1$ and `state` is $\ell$, then
586        `rowDistSims`[`row`][`state`] is denoted by $\theta_{m\ell}^*
587        = u_m^*(\{\ell\})$ in Section 9 of the Paper.
588        '''
589        return tuple(self._rowDistSims)

592    def totalWeight(self):
593        '''`float`: The total weight of the simulation. Denoted by $V$
594        in Section 9 of the Paper.
595        '''
596        return self._totalWeight

434    def _makeSample(self, state):

440        # \[

491        self._bandwidth = samplePDF.factor
492

495            (numRows / (colConc + numRows)) * samplePDF(t)
496        )

522        baseMeas = np.array(idpModel.baseMeas)

524        likelihoods = idpModel.likelihoods
525
526        for m, counts in enumerate(idpModel.rowCounts):
527            # build `rowWeights[m]`
528            self._rowWeights.append([])

533                        self._rowDistSims[i][1] ** counts[1]
534                    )
535                else:
536                    weight = self._rowDistSims[i][0] ** counts[0]
537                    for state in range(1, numStates):
538                        weight *= self._rowDistSims[i][state] ** counts[state]
539                    self._rowWeights[m].append(weight)
540            self._rowWeights[m].append(colConc * likelihoods[m])

542            # build `rowDistSims[m]`
543            choice = choices(range(m + 1), weights=self._rowWeights[m])[0]
544            if choice < m:
545                self._rowDistSims.append(self._rowDistSims[choice])
546            else:
